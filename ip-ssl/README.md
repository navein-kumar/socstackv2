# SOC Stack (IP-SSL) — IP-Based Self-Signed SSL Deployment

A fully automated Docker-based SOC stack designed for **internal/lab/air-gapped deployments** where no domain name or Let's Encrypt SSL is available. Uses **self-signed SSL certificates** with the server's IP address and **dedicated ports per service** via nginx reverse proxy.

> **Use this when:** you don't have a domain name, or deploying on a private/internal network (192.168.x.x, 10.x.x.x), or any public IP without DNS records.
>
> **Use `domain-ssl/` instead when:** you have a domain name with DNS records pointing to your server.

---

## Architecture

```
Client Browser
     │
     ▼ (import ca.crt once to trust self-signed SSL)
     │
     ├── https://IP:8443  →  Nginx  →  Wazuh Dashboard  (SIEM)
     ├── https://IP:8444  →  Nginx  →  Keycloak          (SSO/IAM)
     ├── https://IP:8445  →  Nginx  →  n8n               (Automation)
     ├── https://IP:8446  →  Nginx  →  MISP              (Threat Intel)
     ├── https://IP:8447  →  Nginx  →  TheHive           (Case Mgmt)
     └── https://IP:8448  →  Nginx  →  Cortex            (Analysis)

Direct ports (not proxied):
     ├── IP:1514  → Wazuh Agent (TCP)
     ├── IP:1515  → Wazuh Agent Enrollment
     ├── IP:514   → Syslog (UDP)
     └── IP:55000 → Wazuh Manager API
```

## Directory Structure

```
ip-ssl/
├── .env.example              ← Copy to .env and fill in SERVER_IP

│

│   ├── wazuh/

│   └── n8n/                  ← n8n workflow templates
│
├── data/                     ← Persistent data volumes (auto-created)
└── old/                      ← Previous script versions
```

## Prerequisites

- **OS**: Ubuntu 22.04 / Debian 12 / RHEL 9 (Linux only)
- **RAM**: 16 GB minimum (32 GB recommended)
- **Disk**: 100 GB free
- **Docker**: v24+ with Docker Compose v2
- **OpenSSL**: Installed (for cert generation — comes pre-installed on most distros)
- **No DNS records needed** — works with any IP (private or public)

## Deployment Steps

### Step 1: Configure .env

```bash
cp .env.example .env
nano .env
```

Required fields:
```env
SERVER_IP=192.168.1.100        # Your server's IP address (private or public)
# DEPLOY_DIR is optional — defaults to the folder where the stack files are placed
# DEPLOY_DIR=/opt/socstack
```

Change all `ChangeMe_*` passwords. The IP is used to generate the SSL certificate SAN — it must be reachable from your browser.

---

### Step 2: Copy files to server

```bash
# Copy to any folder you like — the stack works from any location
scp -r ip-ssl/* root@YOUR_SERVER_IP:/opt/socstack/
# Or: scp -r ip-ssl/* root@YOUR_SERVER_IP:/home/user/soc-stack/
```

---

### Step 3: Run pre-deploy checks

```bash
ssh root@YOUR_SERVER_IP
cd /your/deploy/folder          # wherever you copied the files
chmod +x pre-deploy.sh
sudo ./pre-deploy.sh
```

**What pre-deploy does:**
- Checks Docker, RAM, disk space, and port availability
- **Generates self-signed CA + server SSL certificate** for your `SERVER_IP` (with IP SAN)
- Generates Wazuh internal TLS certificates
- Creates all required data directories
- Fixes CRLF line endings (Windows to Linux)

> The `certs/ca.crt` file is created during this step.

---

### Step 4: Start the stack

```bash
cd /your/deploy/folder
docker compose up -d
```

Wait 2–3 minutes for all services to initialize (especially Cassandra and Keycloak).

---

### Step 5: Run post-deploy configuration

```bash
python3 post-deploy.py           # from inside your deploy folder
# or: python3 /your/deploy/folder/post-deploy.py
```

**What post-deploy does automatically:**

| Step | Action |
|------|--------|
| 1    | n8n owner account (disables public signup) |
| 2    | Cortex: DB migration, superadmin, org, API key |
| 3    | TheHive: admin password, organization, analyst user |
| 4    | MISP ↔ TheHive connector |
| 4b   | MISP: load 90+ feeds, enable all, initial cache |
| 5    | Keycloak: SOC realm, OIDC client, groups, SSO users |
| 6    | Wazuh indexer: securityadmin applies OIDC config |
| 7    | Wazuh API: SSO role mapping (run_as) |
| 8    | Save all credentials to `.env.deployed` |

---

### Step 6: Import CA certificate into browser

Before accessing any service, import the self-signed CA into your browser or OS trust store:

**Copy from server:**
```bash
scp root@YOUR_SERVER_IP:/your/deploy/folder/certs/ca.crt ./soc-ca.crt
```

**Windows (Chrome/Edge):**
```powershell
certutil -addstore "Root" soc-ca.crt
```

**Linux:**
```bash
sudo cp soc-ca.crt /usr/local/share/ca-certificates/soc-ca.crt
sudo update-ca-certificates
```

**macOS:**
```bash
sudo security add-trusted-cert -d -r trustRoot -k /Library/Keychains/System.keychain soc-ca.crt
```

**Firefox (uses its own store):**
1. `Preferences → Privacy & Security → View Certificates → Authorities → Import`
2. Select `soc-ca.crt`, tick "Trust this CA to identify websites"

## SSO Login (Wazuh Dashboard)

Three SSO groups are pre-configured in Keycloak:

| Group | Wazuh Access | Users |
|-------|-------------|-------|
| `soc-admin` | Full admin (administrator + users_admin + agents_admin + cluster_admin) | SSO_ADMIN_EMAIL |
| `soc-analyst` | Full admin (same as soc-admin) | SSO_ANALYST_EMAIL |
| `soc-readonly` | Read-only (readonly + agents_readonly + cluster_readonly) | SSO_READONLY_EMAIL |

OpenSearch indexer role mapping (`roles_mapping.yml`):
- `soc-admin` + `soc-analyst` → `all_access` (OpenSearch)
- `soc-readonly` → `wazuh_user` (OpenSearch)

## Manual Post-Deploy Steps (UI)

These require manual configuration in the web UI:

### A. TheHive — Add Cortex & MISP (if API auto-config failed)
1. **Platform Management → Cortex Servers → Add**
   - URL: `http://socstack-cortex:9001`
   - API Key: (from `.env.deployed` → CORTEX_API_KEY)
   - Disable certificate check: Yes
2. **Platform Management → MISP Servers → Add**
   - URL: `https://socstack-misp-core:443`
   - API Key: (from `.env.deployed` → MISP_API_KEY)
   - Skip SSL: Yes

### B. Cortex — Enable Analyzers
- Organization → Analyzers → Refresh → Enable the analyzers you need

### C. n8n — Import Alert Workflow
1. Go to `https://YOUR_IP:8445`
2. New Workflow → Import from file → select `configs/n8n/1_Wazuh_Email_Alert.json`
3. Update Redis node: Hostname = `socstack-n8n-redis`
4. Copy the webhook URL, update `wazuh_manager.conf` hook_url, restart Wazuh Manager

## Troubleshooting

**Browser shows SSL error (NET::ERR_CERT_AUTHORITY_INVALID)**
→ Import `certs/ca.crt` into your OS/browser trust store (see Step 6)

**Keycloak redirects to wrong URL after SSO login**
→ Ensure `SERVER_IP` in `.env` matches the IP you're accessing the server from

**SSO login fails with "Unauthorized"**
→ Run post-deploy again: `python3 post-deploy.py` (from your deploy folder)
→ Check Wazuh indexer OIDC config was applied by securityadmin (Step 6)

**Services not starting**
→ Check RAM (need 16GB+): `free -h`
→ Check vm.max_map_count: `sysctl vm.max_map_count` (needs 262144)
→ Check logs: `docker compose logs SERVICE_NAME`

**Can't reach services from another machine**
→ Ensure firewall allows ports 8443-8448: `ufw allow 8443:8448/tcp`
→ If using AWS/GCP/Azure, add inbound rules for ports 8443-8448

**Regenerate SSL certs for a new IP**
→ Delete `certs/` directory and re-run `sudo ./pre-deploy.sh`
→ Then restart nginx: `docker compose restart socstack-nginx`

## Security Notes

- `certs/ca.key` — CA private key. Keep this secret. Do not share or commit.
- The self-signed CA is only for internal/lab use. For production, use a public CA.
- All services communicate internally over Docker network — only nginx ports are exposed.
- Wazuh agent ports (1514/1515) are exposed directly (not proxied).
